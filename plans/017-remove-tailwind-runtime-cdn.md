## Why this matters

Every public HTML page executes inline Tailwind config and downloads `https://cdn.tailwindcss.com` in the browser. That adds a third-party runtime dependency to page rendering, increases latency, and forces any future Content-Security-Policy to allow remote scripts and inline scripts. The app already ships hand-written CSS in `STYLES`; removing the Tailwind runtime makes pages more deterministic and shrinks the browser security surface.

## Steps

### Step 1: Replace Tailwind utility dependence with static CSS classes

In `src/views/layout.rs`:

- Delete `TAILWIND_CONFIG`.
- Remove both `<script>{}</script>` and `<script src="https://cdn.tailwindcss.com"></script>` from the page shell.
- Replace Tailwind utility-heavy class attributes on `body`, wrapper `div`, `header`, header inner `div`, brand link, tags link, `main`, and `footer` with small semantic classes such as `site-body`, `site-shell`, `site-header`, `site-header-inner`, `site-brand`, `site-nav`, `site-main`, and `site-footer`.
- Add equivalent declarations to `STYLES` so the visual layout remains centered at the current `max-w-3xl` width, keeps the top accent border, supports dark mode, and preserves mobile padding.

Do not change the generated document body content passed as `main`.

**Verify**: `cargo fmt --check` -> exit 0.

### Step 2: Update layout tests to forbid script runtime

In `tests/view_layout_contract.rs`, replace `render_page_includes_tailwind_browser_build` with a test that asserts:

- no `https://cdn.tailwindcss.com`
- no `tailwind.config`
- no `<script` tag in the page shell unless JSON-LD is explicitly present
- the semantic shell classes from Step 1 are present

Keep `render_page_emits_valid_html_attributes`, updating class assertions to match the new semantic shell.

**Verify**: `cargo test --all` -> exit 0.

### Step 3: Run lint

Run the project lint command after tests pass.

**Verify**: `cargo clippy --all-targets --all-features -- -D warnings` -> exit 0.
