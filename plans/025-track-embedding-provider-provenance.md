## Why this matters

When `VOYAGE_API_KEY` is absent, the server persists deterministic mock
embeddings. When the key is later configured, the server switches to real
Voyage query embeddings, but `note_chunks` has no provider/model metadata. That
means real Voyage query vectors can be compared against old mock vectors, giving
arbitrary related-note and `/ask` results until every note is saved again.

Embedding rows need provenance so retrieval only compares compatible vectors,
and operators need an explicit way to rebuild embeddings after changing
providers or models.

## Current state

- Provider selection silently changes vector semantics by configuration:

```rust
// src/ai/mod.rs:152-165
pub fn build_embedder(config: &crate::config::Config) -> Arc<dyn Embedder> {
    match config.voyage_api_key.as_deref() {
        Some(key) => match voyage::VoyageEmbedder::new(key.to_string()) {
            Ok(embedder) => Arc::new(embedder),
            Err(error) => {
                tracing::warn!(%error, "VoyageEmbedder init failed; using mock embedder");
                Arc::new(MockEmbedder)
            }
        },
        None => {
            tracing::info!("VOYAGE_API_KEY not set; using deterministic mock embedder");
            Arc::new(MockEmbedder)
        }
    }
}
```

## Steps

### Step 1: Add provider identity to the embedder contract

Extend the `Embedder` trait in `src/ai/mod.rs` with identity methods:

```rust
fn provider(&self) -> &'static str;
fn model(&self) -> &str;
```

Implement:

- `MockEmbedder`: provider `mock`, model `mock-hash-v1`
- `VoyageEmbedder`: provider `voyage`, model equal to its `model` field

Keep `embed(&self, texts)` unchanged.

**Verify**: `cargo test --lib ai::tests` -> AI unit tests pass.

### Step 2: Add migration 0010 for chunk provenance

Create `migrations/0010_add_note_chunk_embedding_provenance.sql`.

Add columns:

```sql
ALTER TABLE note_chunks
  ADD COLUMN IF NOT EXISTS embedding_provider text NOT NULL DEFAULT 'unknown',
  ADD COLUMN IF NOT EXISTS embedding_model text NOT NULL DEFAULT 'unknown',
  ADD COLUMN IF NOT EXISTS embedding_dimensions integer NOT NULL DEFAULT 1024;

CREATE INDEX IF NOT EXISTS note_chunks_embedding_provenance_idx
  ON note_chunks (embedding_provider, embedding_model, note_id);
```

Update `src/db/migrations.rs` so version numbers stay contiguous and tests know
about migration 0010.

Do not guess provenance for existing rows. Existing rows should become
`unknown`; retrieval will ignore them until reindexed.

**Verify**: `cargo test --lib db::migrations` -> migration tests pass.

### Step 3: Persist provenance on replacement

Change `replace_note_chunks` to accept provider/model/dimensions, or introduce a
small struct:

```rust
pub struct EmbeddingSource<'a> {
    pub provider: &'a str,
    pub model: &'a str,
    pub dimensions: i32,
}
```

Insert those fields with every chunk row. Update `index_note` to pass identity
from `embedder`.

Do not store secrets, endpoint URLs, or API-key presence. Provider/model only.

**Verify**: `cargo clippy --all-targets --all-features -- -D warnings` -> exit 0.

### Step 4: Filter retrieval by active provenance

Update vector retrieval functions so they only compare chunks whose provenance
matches the active embedder:

- `search_chunks`
- `related_notes`
- if Plan 024 already landed, `related_notes_for_note`

Thread `provider` and `model` from `state.embedder` through `src/http/ai.rs`.

Expected behavior:

- if only `unknown` rows exist, vector search returns no hits;
- `/ask` then falls back to FTS, as it already does;
- `/related` returns an empty related list until compatible chunks exist.

Do not fall back to comparing incompatible vectors.

**Verify**: `DATABASE_URL=<postgres-url> cargo test --test ai_contract` -> existing AI tests pass after updating helpers/expectations as needed.

### Step 5: Add a reindex command or STOP

Operators need a way to rebuild chunks after enabling Voyage or changing the
model. Add the narrowest command that fits the current CLI style, for example:

```text
inkwell db reindex-embeddings
```

It should:

- load `Config::from_env()`;
- build the active embedder with `build_embedder`;
- list all documents in bounded pages;
- call `index_note` for each document using its current `id`, `version`, and
  `body_markdown`;
- print counts for indexed, skipped, and failed notes;
- never print document body or secrets.

If this command becomes larger than a focused implementation, STOP and report
instead of burying it inside the provenance change. In that case, complete
Steps 1-4 and create a follow-up plan for reindexing before marking this plan
DONE.

**Verify**: `DATABASE_URL=<postgres-url> cargo run --bin inkwell -- db reindex-embeddings` -> exits 0 and prints summary counts.
